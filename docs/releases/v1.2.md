# MUTX v1.2

MUTX v1.2 is the release where governance and observability become first-class citizens in the operator stack.

This release ships Faramesh as a core governance engine, adds OpenClaw agent observability via the agent-run standard, strengthens the AARM security layer, and expands the CLI with a complete governance command family.

## What v1.2 actually delivers

### 1. Faramesh Governance Engine (Core)

Faramesh is now integrated as MUTX's governance backend — not a plugin, not an external dependency to be managed separately, but a first-class component that ships with MUTX.

Core capabilities:

- **Policy enforcement** via FPL (Faramesh Policy Language): PERMIT, DENY, DEFER outcomes on tool calls
- **Session budgets**: max spend, daily limits, call counts per session
- **Phase workflows**: tool visibility scoped by workflow stage (intake → execution → delivery)
- **Credential brokering**: API key stripping and ephemeral credential injection per call
- **Ambient guards**: rate limiting across sessions, not just per call
- **SPIFFE/SPIRE identity**: workload identity provider for secure agent-to-agent and agent-to-backend authentication

CLI governance family:

```
mutx governance status        # cluster and policy status
mutx governance decisions      # recent decisions
mutx governance pending        # pending approvals
mutx governance metrics        # Prometheus-format metrics
mutx governance tail           # live decision feed
mutx governance start          # start governance supervisor
mutx governance approve <id>   # approve a pending action
mutx governance deny <id>      # deny a pending action
mutx governance kill <id>      # terminate an active session
mutx governance policy         # policy management
mutx governance credential     # credential broker status
mutx governance webhook        # webhook configuration
mutx governance supervise      # supervision controls
```

### 2. Observability Dashboard

Agent run tracking is now available in the dashboard via the [agent-run](https://github.com/builderz-labs/agent-run) open standard.

MUTX Schema (derived from agent-run):

- **MutxRun** — toplevel run record (replaces agent_run table)
- **MutxStep** — individual steps within a run
- **MutxCost** — token and cost tracking
- **MutxContext** — accumulated context window
- **MutxFeedback** — human feedback on runs

The dashboard exposes run history, step traces, cost aggregation, and session-level feedback under `/dashboard/observability`.

Python SDK: `sdk/mutx/observability.py` provides `OpenClawObservability` for reporting agent runs.

### 3. AARM Security Layer

The [Autonomous Action Runtime Management](https://github.com/aarm-dev/docs) specification is now implemented as MUTX's security evaluation layer:

- **Action Mediation** — intercepts and evaluates actions before execution
- **Context Accumulator** — builds security context from session history
- **Policy Engine** — evaluates against loaded AARM policies
- **Approval Service** — handles human-in-the-loop approvals
- **Receipt Generator** — produces audit receipts for approved actions
- **Telemetry Exporter** — exports security telemetry

### 4. Credential Broker

Multi-backend secret management now covers:

- HashiCorp Vault
- AWS Secrets Manager
- GCP Secret Manager
- Azure Key Vault
- 1Password
- Infisical

### 5. Textual TUI — Governor and Observability Tabs

The `mutx tui` operator shell now includes dedicated tabs for:

- **Governor tab**: governance status, pending decisions, live metrics tail
- **Observability tab**: agent run tracking and session history

TUI loads governance and observability data on mount, giving operators a terminal-native view into what the dashboard provides in the browser.

### 6. `mutx update` Command

MUTX can now update itself from git without reinstallation:

```bash
mutx update
```

### 7. Homebrew Tap Improvements

The Homebrew formula at `homebrew-tap/Formula/mutx.rb` now includes:

- Non-network `mutx status` formula test guidance
- `cli-vX.Y.Z` tag tracking aligned with root `pyproject.toml`

### 8. CI/CD Improvements

- Ansible playbooks fixed with FQCN module names (`ansible.builtin.*`)
- YAML truthy values corrected (`yes`/`no` → `true`/`false`)
- Node.js 24 in GitHub Actions (replacing deprecated Node.js 20)
- Python tests now cover observability routes, RAG search (503 with feature flag), scheduler (503 with feature flag)

| Feature | Status | Notes |
|---------|--------|-------|
| RAG Search | Preview | Returns 503 with `X-Feature-Flag: rag.search` until OpenAI key present |
| Scheduler | Preview | Returns 503 with `X-Feature-Flag: scheduler` until runtime is configured |
| Shell pages (channels, skills, orchestration) | Placeholder | UI stubs, not yet wired to backend |
