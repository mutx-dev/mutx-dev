# MUTX v1.3

MUTX v1.3 is the first release that turns MUTX into a coherent operator lane instead of a collection of half-promoted surfaces.

This release makes the macOS app the primary operator UI, publishes a real `mutx.dev/releases` summary for the signed artifact set, promotes `app.mutx.dev/dashboard` as the supported browser shell for stable routes, keeps `app.mutx.dev/control/*` honestly preview, and hardens the release path around first-party download routes, fail-closed validation, and production promotion on Railway.

## What v1.3 ships

### Desktop and download lane

- first-party desktop download routes at `mutx.dev/download/macos/*`
- a public release summary at `mutx.dev/releases`
- signed and notarized macOS artifact lane published from GitHub Releases
- Apple Silicon as the primary download lane, with Intel still available
- release notes published on the synced docs surface instead of only on GitHub

### Supported operator surfaces

- `mutx.dev` as the public product and install surface
- `mutx.dev/releases` as the public release summary and artifact map
- `docs.mutx.dev` as the canonical documentation and API narrative
- `app.mutx.dev/dashboard` as the supported browser operator shell for stable routes
- `app.mutx.dev/control/*` kept reachable but explicitly preview and out of primary stable navigation

### Release hardening

- fail-closed release validation for lint, typecheck, build, generated-artifact verification, serial browser smoke, desktop cockpit smoke, and signed desktop validation
- a single first-view dashboard contract behind `GET /api/dashboard/overview`
- release-facing version truth aligned to `1.3.0` across app, CLI distribution, SDK, release notes, and smoke fixtures
- GitHub-synced docs plus Railway production promotion runbooks so the launch does not depend on local tribal knowledge
- manual DMG assembly plus mounted-app signature verification so the GitHub release carries valid notarized artifacts for both macOS architectures

## Stable versus preview

### Supported in v1.3

- `mutx.dev`
- `mutx.dev/releases`
- `docs.mutx.dev`
- the macOS operator app downloaded from `mutx.dev/download/macos`
- `app.mutx.dev/dashboard` for stable browser operator routes
- the CLI install lane via `curl -fsSL https://mutx.dev/install.sh | bash`

### Still preview in v1.3

- `app.mutx.dev/control/*`
- preview-labeled or redirect-backed dashboard routes kept out of primary stable navigation
- runtime-gated backend capabilities such as scheduler and RAG search until their live contracts are complete

## Install and artifact sources

- public release summary: `https://mutx.dev/releases`
- first-party download origin: `https://mutx.dev/download/macos`
- public release notes: `https://docs.mutx.dev/docs/releases/v1.3`
- signed artifacts and checksums: `https://github.com/mutx-dev/mutx-dev/releases`
- stable dashboard entry: `https://app.mutx.dev/dashboard`

## Validation bar

The shipped release is built to pass:

- `npm run release:check`
- signed desktop validation on the release machine
- Railway production promotion for the frontend and backend services after the RC closes
- post-promote smoke on `mutx.dev`, `app.mutx.dev/dashboard`, `api.mutx.dev/health`, `api.mutx.dev/ready`, and this docs page after GitBook sync

## What changed operationally

1. The GitHub release now carries signed and notarized Apple Silicon and Intel desktop artifacts plus a checksum file.
2. `mutx.dev/download/macos` resolves directly to the current stable desktop assets.
3. `mutx.dev/releases` gives the public website a real release summary instead of forcing users straight into GitHub or docs.
4. `docs/releases/v1.3.md` remains the repo-backed canonical narrative for what is supported and what is still preview.
5. `app.mutx.dev/dashboard` is the supported browser operator shell for stable routes, while the control demo remains preview.

For the fail-closed release gate, see [v1.3 Release Checklist](./v1.3-checklist.md).
