**Human approval gate ‚Äî button format (2026-03-30 15:25):**

Discord v2 components ‚Äî CORRECT format (from official docs):
```json
{
  "reusable": false,
  "blocks": [
    {
      "type": "actions",
      "buttons": [
        {"label": "POST", "style": "success", "allowedUsers": ["804823236222779413"]},
        {"label": "RETRY", "style": "secondary"},
        {"label": "CANCEL", "style": "danger"}
      ]
    }
  ]
}
```
Key: NO `custom_id`, `callbackData`, or `internalCustomId`. Button LABEL is what gets sent back on click.
`style`: success | secondary | danger | primary | link
`allowedUsers`: Discord user IDs who can click (optional, restrict to owner)

Flow: button click ‚Üí label text ("POST"/"RETRY"/"CANCEL") routes back to Cypher as message ‚Üí `approval_dispatcher.sh` maps to yes/retry/cancel ‚Üí CDP daemon acts.
