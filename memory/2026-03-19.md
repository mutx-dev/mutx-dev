
## 2026-03-19 11:51 Europe/Rome — OpenClaw/MUTX audit and strategic simplification
- Fortune explicitly decided to switch to Codex full-time for pushing MUTX forward because the OpenClaw autonomy/fleet setup created too much complexity.
- Durable audit conclusion: the biggest source of future agent confusion is not code quality but **environment legibility** — too many overlapping worktrees, stale state files, half-retired cron/company roles, ACP/session drift, and imported agent noise.
- Key structural findings from direct inspection:
  - OpenClaw config still runs with `sandbox.mode=off`, `tools.exec.security=full`, and `tools.exec.ask=off`, making the old fleet unusually powerful and hard to reason about.
  - The 12-role company/cron model became mostly orchestration overhead compared with the direct-to-main `live-main` UI lane, which was the clearest successful execution pattern.
  - Current workspace topology is confusing: canonical-looking work exists across `~/.openclaw/workspace`, `~/.openclaw/workspace/.worktrees/*`, and `~/mutx-worktrees/factory/*` (including `live-main`, `backend`, `frontend`, `ship`, `auditor`, `acp-swarm/*`, and dozens of `issue-*` worktrees).
  - Multiple state surfaces now conflict or drift (`mutx-fleet-state.md`, `mutx-fleet-bible.md`, `reports/company/ROSTER.md`, worker-state files, cron state, memory notes), so future agents cannot safely assume “read state first” means one truthful source.
  - ACP persistent-thread/session ambitions were not reliable from the current chat surface; session binding and ACP metadata drift created stale/uncontrollable sessions.
  - Discord voice slowness/non-responsiveness was confirmed to be structural/runtime-level (voice lifecycle + command deployment churn), not caused by model thinking; thinking was already off by default.
- Durable recommendation from the audit: simplify aggressively around one canonical Codex-driven worktree/repo path, retire the company-role/cron theater, mark stale files as historical, and create one short “current operating mode” source of truth so future agents know what is live vs retired.
