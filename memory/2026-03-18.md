## openclaw-control-ui deployment bug (18:25 UTC)

- Repeated production/runtime failures were traced to schema drift in `agent_logs`: application code tries to insert `meta_data`, but the live Postgres table does not have that column.
- Symptom pattern: background agent monitor marks stale `test-agent` as FAILED every cycle, then crashes on `INSERT INTO agent_logs (... meta_data ...)` with `asyncpg.exceptions.UndefinedColumnError`.
- This caused repeated monitor errors and container restarts even though deployments themselves were otherwise succeeding.
- Durable diagnosis: there is a migration/model mismatch around `agent_logs.meta_data`; next debugging pass should compare SQLAlchemy model vs Alembic migrations vs live DB schema before touching monitor logic.
